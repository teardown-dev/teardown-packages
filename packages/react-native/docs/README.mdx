# Teardown React Native SDK Documentation

Complete documentation for the Teardown SDK for React Native and Expo applications.

## Overview

The Teardown SDK provides a comprehensive solution for managing device identity, force updates, logging, and analytics in React Native and Expo applications. Built with TypeScript, it offers a type-safe, modular architecture designed for production use.

## Features

- 🔐 **Device & User Identity** - Unique device fingerprinting and user session management
- 🔄 **Force Updates** - Automatic version checking with optional or required update flows
- 📱 **Device Information** - Comprehensive device, OS, and app information collection
- 💾 **Storage** - Namespaced persistent storage with platform adapters
- 📝 **Logging** - Structured logging system with debug modes
- ⚡ **Performance** - Optimized with caching, throttling, and efficient state management
- 🎯 **Type Safety** - Full TypeScript support with runtime validation
- 🧪 **Testable** - Easy to mock and test

## Quick Start

### 1. Initialize the SDK

```tsx
import { TeardownCore, ExpoDeviceAdapter } from '@teardown/react-native';
import * as Device from 'expo-device';
import * as Application from 'expo-application';
import AsyncStorage from '@react-native-async-storage/async-storage';

const teardown = new TeardownCore({
  api: {
    api_key: 'your-api-key',
    org_id: 'your-org-id',
    project_id: 'your-project-id',
    environment_slug: 'production',
  },
  storage: {
    adapter: {
      getItem: (key) => AsyncStorage.getItem(key),
      setItem: (key, value) => AsyncStorage.setItem(key, value),
      removeItem: (key) => AsyncStorage.removeItem(key),
    },
  },
  device: {
    adapter: new ExpoDeviceAdapter({ Device, Application }),
  },
  identity: {
    identifyOnLoad: true,
  },
  forceUpdate: {
    throttleMs: 30000,
    checkCooldownMs: 300000,
  },
});
```

### 2. Wrap Your App

```tsx
import { TeardownProvider } from '@teardown/react-native';

export default function App() {
  return (
    <TeardownProvider core={teardown}>
      <YourApp />
    </TeardownProvider>
  );
}
```

### 3. Use the SDK

```tsx
import { useTeardown, useForceUpdate } from '@teardown/react-native';

function YourComponent() {
  const { core } = useTeardown();
  const { isUpdateRequired } = useForceUpdate();

  const handleLogin = async () => {
    const result = await core.identity.identify({
      user_id: 'user-123',
      email: 'user@example.com',
    });
    
    if (result.success) {
      console.log('User identified');
    }
  };

  if (isUpdateRequired) {
    return <ForceUpdateScreen />;
  }

  return <YourContent />;
}
```

## Core Concepts at a Glance

### Session States

```typescript
type SessionState = 
  | { type: 'unidentified' }
  | { type: 'identifying' }
  | { type: 'identified', session: Session, version_status: VersionStatus }
```

### Version Status

```typescript
type VersionStatus =
  | { type: 'unknown' }
  | { type: 'checking' }
  | { type: 'up_to_date' }
  | { type: 'update_available' }
  | { type: 'update_required' }
```

### Error Handling

```typescript
type AsyncResult<T> = 
  | { success: true, data: T }
  | { success: false, error: string }
```

## Common Use Cases

### User Login

```tsx
const { core } = useTeardown();

const handleLogin = async (email: string, password: string) => {
  const user = await yourAuthService.login(email, password);
  
  const result = await core.identity.identify({
    user_id: user.id,
    email: user.email,
    name: user.name,
  });
  
  if (result.success) {
    navigation.navigate('Home');
  }
};
```

### Force Update Screen

```tsx
function App() {
  const { isUpdateRequired } = useForceUpdate();
  
  if (isUpdateRequired) {
    return (
      <View>
        <Text>Update Required</Text>
        <Button title="Update Now" onPress={openAppStore} />
      </View>
    );
  }
  
  return <YourApp />;
}
```

### Device Info Collection

```tsx
const { core } = useTeardown();

const deviceInfo = await core.device.getDeviceInfo();
console.log(`Device: ${deviceInfo.hardware.device_name}`);
console.log(`OS: ${deviceInfo.os.name} ${deviceInfo.os.version}`);
```

### Custom Storage

```tsx
const { core } = useTeardown();
const storage = core.storage.createStorage('settings');

storage.setItem('theme', 'dark');
const theme = storage.getItem('theme');
```

### Structured Logging

```tsx
const logger = core.logging.createLogger({ 
  name: 'MyFeature',
  debugEnabled: __DEV__ 
});

logger.info('Operation started');
logger.debug('Processing item', item);
logger.error('Operation failed', error);
```

## TypeScript Support

The SDK is written in TypeScript and provides full type definitions:

```tsx
import type {
  TeardownCore,
  SessionState,
  VersionStatus,
  DeviceInfo,
  AsyncResult,
} from '@teardown/react-native';
```

## Browser Compatibility

When using web adapters:
- Modern browsers (Chrome, Firefox, Safari, Edge)
- localStorage or IndexedDB support required
- Service Worker support for push notifications (optional)

## Requirements

- React 16.8+ (hooks support)
- React Native 0.60+
- TypeScript 4.5+
- Expo SDK 45+ (if using Expo)
