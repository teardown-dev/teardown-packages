# Force Updates

Managing app version requirements and force update flows with Teardown SDK.

## Overview

The ForceUpdateClient provides automatic version checking and enforcement:
- Automatic background version checks
- Configurable check frequency and throttling
- Integration with app foreground/background events
- Version status persistence across app sessions
- React hook for UI integration

## Version Statuses

```typescript
type VersionStatus =
  | { type: "unknown" }          // Initial state, not yet checked
  | { type: "checking" }         // Check in progress
  | { type: "up_to_date" }       // Current version is latest
  | { type: "update_available" } // Update available but optional
  | { type: "update_required" }  // Update required to continue
```

## Configuration

```typescript
export const teardown = new TeardownCore({
  // ...
  forceUpdate: {
    // Minimum time between foreground checks (default: 30s)
    throttleMs: 30000,
    
    // Cooldown after successful check (default: 5min)
    checkCooldownMs: 300000,
    
    // Check version for anonymous devices (default: false)
    identifyAnonymousDevice: false,
  },
});
```

## Automatic Checking

Version checks happen automatically when:
1. User is identified (via `identity.identify()`)
2. Session is refreshed (via `identity.refresh()`)
3. App comes to foreground (with throttling and cooldown)

### Foreground Check Logic

```typescript
// Check only if:
// 1. Enough time since last foreground (throttleMs)
// 2. Enough time since last check (checkCooldownMs)

App Foreground → Check Throttle → Check Cooldown → Identify/Refresh
```

## React Hook

Use the `useForceUpdate` hook to integrate version status into your UI:

```typescript
import { useForceUpdate } from "@teardown/react-native";

function App() {
  const { versionStatus, isUpdateAvailable, isUpdateRequired } = useForceUpdate();

  if (isUpdateRequired) {
    return <ForceUpdateScreen />;
  }

  if (isUpdateAvailable) {
    return (
      <>
        <UpdateBanner />
        <MainApp />
      </>
    );
  }

  return <MainApp />;
}
```

### Hook Return Type

```typescript
export type UseForceUpdateResult = {
  /** Current version status */
  versionStatus: VersionStatus;
  
  /** True if update is available but not required */
  isUpdateAvailable: boolean;
  
  /** True if update is required (blocking) */
  isUpdateRequired: boolean;
};
```

## Manual Version Checking

### Check Current Status

```typescript
const status = core.forceUpdate.getVersionStatus();

```

### Listen to Status Changes

```typescript
const unsubscribe = core.forceUpdate.onVersionStatusChange((status) => {
  console.log("Version status changed:", status.type);
  
  if (status.type === "update_required") {
    // Show force update screen
  }
});

// Clean up
unsubscribe();
```

### Manual Check

Trigger a version check by identifying or refreshing:

```typescript
// For authenticated users
await core.identity.refresh();

// For anonymous users (if identifyAnonymousDevice is true)
await core.identity.identify();
```

## UI Patterns

### Blocking Update Screen

Show a full-screen takeover when update is required:

```typescript
import { useForceUpdate } from "@teardown/react-native";

export function ForceUpdateScreen() {
  const { versionStatus } = useForceUpdate();

  if (versionStatus.type !== "update_required") {
    return null;
  }

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Update Required</Text>
      <Text style={styles.message}>
        A new version of the app is available. Please update to continue.
      </Text>
      <Button
        title="Update Now"
        onPress={() => {
          const storeUrl = Platform.select({
            ios: "https://apps.apple.com/app/yourapp",
            android: "https://play.google.com/store/apps/details?id=com.yourapp",
          });
          Linking.openURL(storeUrl);
        }}
      />
    </View>
  );
}
```

### Optional Update Banner

Show a dismissible banner for optional updates:

```typescript
export function UpdateBanner() {
  const { isUpdateAvailable } = useForceUpdate();
  const [dismissed, setDismissed] = useState(false);

  if (!isUpdateAvailable || dismissed) {
    return null;
  }

  return (
    <View style={styles.banner}>
      <Text>A new version is available</Text>
      <Button title="Update" onPress={handleUpdate} />
      <Button title="Later" onPress={() => setDismissed(true)} />
    </View>
  );
}
```

### App Root Integration

```typescript
export default function RootLayout() {
  return (
    <TeardownProvider core={teardown}>
      <ForceUpdateGate>

          <Stack />
        </SafeAreaProvider>

function ForceUpdateGate({ children }) {

  }

  return children;
}
```

```typescript
const sessionState = core.identity.getSessionState();

// App comes to foreground

// → API returns version status
```


## Throttling & Cooldowns

Prevent excessive API calls with throttling and cooldowns:

```typescript
{
  forceUpdate: {
    // User opens/closes app rapidly
    throttleMs: 30000, // Wait 30s between foreground checks
    
    // Avoid hitting API too often
    checkCooldownMs: 300000, // Wait 5min since last successful check
  }
}
```

### Example Timeline

```
T=0:00  - App foreground → Check (allowed)
T=0:10  - App background
T=0:15  - App foreground → Check (blocked by throttle, < 30s)
T=0:40  - App foreground → Check (blocked by cooldown, < 5min)
T=5:30  - App foreground → Check (allowed, past both limits)
```

## Version Management

Version status is determined by your Teardown dashboard configuration:
- **Minimum Version**: Versions below this require update
- **Recommended Version**: Versions below this show update available
- **Latest Version**: Current version in app stores

### Status Resolution

```
User Version < Minimum → update_required
User Version < Recommended → update_available
User Version >= Recommended → up_to_date
```

## Persistence

Version status is persisted across app sessions:

```typescript
// First launch, after version check
versionStatus.type === "update_available"

// App restart
const status = core.forceUpdate.getVersionStatus();
// status.type === "update_available" (restored from storage)
```

## Best Practices

1. **Place at Root**: Wrap ForceUpdateGate at app root level
2. **Block Navigation**: Prevent navigation when update required
3. **Store Links**: Use deep links to app store pages
4. **Clear Messaging**: Explain why update is needed
5. **Test Throttling**: Verify throttling prevents excessive checks
6. **Handle States**: Account for all version status states
7. **Track Events**: Log version check events for monitoring

## Troubleshooting

### Checks Not Happening

- Verify `identifyAnonymousDevice: true` for anonymous users
- Check throttleMs and checkCooldownMs values
- Ensure app is foregrounding (not just reopening)

### Status Not Updating

- Check network connectivity
- Verify API credentials are correct
- Look for errors in logs (enable debug mode)

### Infinite Update Loops

- Ensure version numbers are incremented properly
- Verify dashboard configuration matches deployed version
- Check for version comparison bugs in custom logic
